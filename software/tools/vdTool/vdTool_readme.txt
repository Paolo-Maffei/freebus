Das vdTool analysiert eine vd_ Datei und stellt das Ergebnis in lesbarer Form dar. Als input File muß man die entpackte .vd_ Datei verwenden. An die kommt man dran, indem man die .vd1 Datei in ETS importiert und währenddessen im database Verzeichnis die temporär angelegte .vd_ kopiert. Eleganter geht es natürlich, wenn man die .vd1 in .zip umbenennt und die .vd_ direkt extrahiert. Dazu muss man das Passwort kennen.

Nach dem öffnen wird eine Datenbank (.db_) angelegt. Diese kann man sich bei Bedarf mit jedem beliebigen sqlite tool anschauen. Im Textfenster steht dann das Ergebnis. Damit man beim zweiten Mal nicht wieder alles konvertieren muß, kann man auch die .db_ direkt öffnen.
